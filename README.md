# PromptCade: Generative Computer Vision Engine

**PromptCade** is a next-generation web engine that transforms plain English prompts into fully playable, 60fps computer-vision games. You don't need a controller—your physical body is the controller.

Type *"A game where I have to physically punch falling zombie heads to survive"* and the engine will instantly compile a game with physical hitboxes, zombie targets, and real-time AI skeleton tracking.

## 🚀 The Vision
What if "Vibe Coding" applied to physical gameplay? 
Most generative AI applications output text, images, or static code. PromptCade introduces a **Parametric Translation Engine** that maps natural language directly into physical physics paradigms.

## 🎮 How It Works

### For the Player (Non-Technical)
1. **Prompt**: Type any physical game idea you can imagine. 
2. **Compile**: The AI instantly understands the physics required (Dodging, Striking, Counting Reps, Holding Poses) and the visual theme.
3. **Play**: Step back from your webcam. An AI skeleton maps your joints in real-time, allowing you to physically jump, squat, punch, and dodge to interact with the game world.

### For the Engineer (Technical)
PromptCade is built to guarantee **0 latency and 100% stability**, bypassing the traditional pitfalls of live AI code generation.

* **The Vision Layer (`MediaPipe Pose`)**: Runs completely client-side. We extract 33 3D landmarks of the human body at 60fps.
* **The Physics Loop (`requestAnimationFrame`)**: High-frequency body tracking destroys React's render cycle. We built a custom native `requestAnimationFrame` loop that directly manipulates the DOM and Canvas, bypassing React state entirely to maintain a locked 60 frames per second.
* **The AI Compiler (`Azure OpenAI`)**: A highly restricted LLM prompt takes the user's string and compiles it into a strict JSON schema (`GameConfig`). It assigns one of our 4 foundational physics mechanics, a primitive body movement, dynamic emoji assets, and hex themes.
* **The Physical Evaluators**: Custom math algorithms calculate movement intent. (e.g. A "jump" isn't triggered by moving your head up; the system mathematically calculates the vertical distance between your physical shoulders and hips to prevent cheating).

## 🧠 Core Physics Mechanics
The engine's vocabulary is currently built on 4 foundational paradigms. From these 4 mechanics, infinite game variations can be generated by the AI:
1. `survival-dodge`: A high-stakes lane engine where physical movement is required to evade dynamic falling hitboxes.
2. `strike-targets`: An interception engine calculating the velocity of wrists/hands to destroy objects upon collision.
3. `count-reps`: A form-tracking engine that validates exact physical exercise metrics (Squats, Jumps, Raises).
4. `pose-match`: A spatial-awareness engine demanding precise joint alignment for a duration of time.

## 🛠️ Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/chinnuteja/motions-prompt-arcade.git
cd motions-prompt-arcade

# 2. Install dependencies
npm install

# 3. Setup your Environment Variables
# Create a .env.local file and add your Azure OpenAI keys
AZURE_OPENAI_API_KEY=your_key
AZURE_OPENAI_ENDPOINT=your_endpoint
AZURE_OPENAI_DEPLOYMENT_NAME=your_deployment

# 4. Start the Engine
npm run dev
```

## 🏗️ Architecture Stack
* **Framework:** Next.js 14+ (App Router)
* **Computer Vision:** Google MediaPipe (Client-side WASM)
* **Generative AI:** Azure OpenAI (`gpt-4o-mini` / `gpt-4o`)
* **Styling:** Vanilla CSS Modules (For extreme performance & DOM manipulation)
